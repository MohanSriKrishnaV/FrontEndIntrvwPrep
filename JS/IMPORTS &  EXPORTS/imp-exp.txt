## üîπ What are `import` and `export` in JS?

They are part of **ES Modules (ESM)** ‚Äî JavaScript‚Äôs built-in module system.

They help you:

* Split your code into multiple files
* Reuse logic cleanly
* Avoid polluting the global scope

---

## üß± 1. **Named Exports**

You explicitly export each variable, function, or class.

```js
// üìÅ mathUtils.js
export const PI = 3.14159;

export function add(a, b) {
  return a + b;
}

export function multiply(a, b) {
  return a * b;
}
```

To import named exports:

```js
// üìÅ app.js
import { PI, add, multiply } from './mathUtils.js';

console.log(add(2, 3)); // 5
console.log(PI); // 3.14159
```

You can also **rename** imports or exports:

```js
// in export
export { add as sum };

// in import
import { sum as addition } from './mathUtils.js';
```

---

## üåü 2. **Default Export**

Every module can have **only one default export**.

```js
// üìÅ logger.js
export default function log(message) {
  console.log(`Log: ${message}`);
}
```

Importing a default export:

```js
// üìÅ app.js
import log from './logger.js'; // name can be anything
log('Hello'); // Log: Hello
```

You can name it whatever you want when importing ‚Äî because it‚Äôs *default*.

---

## ‚ö° 3. **Combining named + default exports**

```js
// üìÅ shapes.js
export const square = n => n * n;
export const circle = r => Math.PI * r * r;

export default function shapeInfo() {
  console.log('Shapes utility module');
}
```

Importing both:

```js
import shapeInfo, { square, circle } from './shapes.js';
shapeInfo();
console.log(square(4));
```

---

## üß© 4. **Import Everything**

You can import all named exports as a single object:

```js
import * as math from './mathUtils.js';

console.log(math.add(2, 3)); // 5
console.log(math.PI);        // 3.14159
```

---

## üåÄ 5. **Re-exporting (Module Aggregation)**

If you want to re-export from another module:

```js
// üìÅ utils/index.js
export * from './mathUtils.js';
export { default as log } from './logger.js';
```

Now you can import from just one file:

```js
import { add, log } from './utils/index.js';
```

---

## üöß 6. **Dynamic Imports (Lazy Loading)**

Sometimes you want to import **only when needed** (useful in Angular, React lazy routes, etc.)

```js
if (someCondition) {
  const { add } = await import('./mathUtils.js');
  console.log(add(2, 3));
}
```

This returns a **Promise** that resolves to the module.

---

## üß† Notes:

| Concept        | Description                                               |
| -------------- | --------------------------------------------------------- |
| Static Import  | `import` at top level (hoisted, happens before code runs) |
| Dynamic Import | `await import()` ‚Äî on demand                              |
| Named Exports  | `{ a, b }`                                                |
| Default Export | single main export                                        |
| Mixing both    | Allowed                                                   |
| File Extension | Must include `.js` (or `.mjs`) when running via Node ESM  |
